##🛠️ IT Admin Guide: The OpenShift Prerequisite Tracker

Hello! If you've been sent this file, it means we are getting ready to build an OpenShift Container Platform environment for you.

As your IT team, we want to make sure this deployment goes as smoothly and quickly as possible. OpenShift is a powerful platform, but it requires a lot of "moving parts" to be perfectly aligned before we can even start building. We need things like IP addresses, DNS records, cloud account details, and server sizes sorted out ahead of time.

Instead of sending you a confusing 20-page Word document, I've created this simple, interactive web tool.

Don't worry if you aren't a computer expert! This guide will walk you through exactly how to use it.

##🤔 What is this file?

The file you downloaded (likely called index.html) is a miniature web application packed into a single file.

You do not need to install any software, create an account, or connect to a database to use it. It runs entirely inside your own web browser (like Chrome, Edge, or Safari).

##🚀 How to Use It (Step-by-Step)

###Step 1: Open the Application

Simply find the index.html file on your computer (likely in your Downloads folder) and double-click it. It will automatically open in your default web browser.

###Step 2: Set Your Environment (Crucial!)

When the page loads, the very first thing you will see is a dark grey box titled "Environment Configuration". This is the brain of the checklist.

Because OpenShift can be installed in many different ways (on Amazon AWS, Microsoft Azure, VMware, etc.), we don't want to overwhelm you with questions that don't apply to you.

Click the checkboxes that match your specific project plan.

Watch as the page magically shrinks or expands! It will only show you the sections you actually need to fill out.

###Step 3: Fill Out the Form

Scroll down and start typing in the answers. You'll see text boxes, checkboxes, and dropdown menus.

Please be as accurate as possible. If you don't know the answer to a specific networking or infrastructure question, please reach out to your internal IT team or Red Hat representative.

###Step 4: Don't Worry About Losing Your Work! (Auto-Save)

Forms can be frustrating, especially if your computer restarts or you accidentally close the tab.

I have built "Auto-Save" into this tool. Every time you type a letter or click a box, your progress is secretly saved directly to your web browser.

If you accidentally close the window, just double-click the index.html file again. All your text will be right where you left it!

###Step 5: Save and Share with Us

Because this tool doesn't connect to the internet, I cannot see what you are typing. Once you have filled out everything to the best of your ability, you need to send it back to us.

Look at the top right of the screen (in the red header bar).

Click the white button that says "Print / PDF".

Your computer's print menu will pop up.

Instead of selecting a physical printer, change the destination to "Save as PDF" (or "Microsoft Print to PDF").

Save the PDF file to your computer, and email that PDF back to your Red Hat implementation team!

##🙋 Frequently Asked Questions

####Is my data secure? Where is it going?
Yes, it is 100% secure. The data never leaves your computer. When you type in this form, the data is saved in your browser's local memory. It is not sent to the cloud, it is not sent to Red Hat, and it is not sent to me. That is why you must "Save as PDF" to share your answers.

####I checked the wrong box and want to start completely over. How do I do that?
At the top right of the page, next to the Print button, there is a small icon of a Trash Can. Clicking this will wipe the form completely clean and delete your saved data, allowing you to start fresh.

####Why does the page look different when I hit Print?
I designed the print view to be as clean as possible. When you hit Print, the application automatically hides the configuration buttons, forcefully opens any collapsed menus, and removes borders so it reads like a professional, official document.

If you run into any issues opening the file or have questions about the prerequisites themselves, please reach out to your designated Red Hat Architect or Project Manager.
